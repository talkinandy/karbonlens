### Added
- **T01 VPS foundation** ([feature/v0.1-impl]) ‚Äî provisioned the Hetzner CX32 box: PostgreSQL 16.13 + PostGIS 3.6.3, `pg_trgm` 1.6 + `pgcrypto` 1.3 extensions in the `karbonlens` database, `karbonlens` Unix user (no-login, uid 999) and Postgres role (scram-sha-256 on loopback only ‚Äî wrong-password rejection verified with `FATAL: password authentication failed` exit 2). Standard directories `/opt/karbonlens` and `/var/{log,lib}/karbonlens` owned by the karbonlens user. Postgres bound to `localhost` only (`listen_addresses = 'localhost'`; no public IP exposure). Secrets at `/root/karbonlens-secrets.txt` (chmod 600, root:root). Runbook at `docs/runbooks/vps-setup.md`; implementation report at `docs/stories/reports/T01-implementation-report.md`. Code-audit verdict: PASS-WITH-FIXES (SHA 5c5bbd8).
- **T02 Schema migration 001** ([feature/T02-schema-migration]) ‚Äî 15 tables (projects, registries, issuances, retirements, idx_monthly_snapshots, satellite_alerts, regulatory_events, project_scores, project_match_queue, users, accounts, sessions, verification_tokens, notifications, schema_migrations) with PostGIS geography columns, `users.email_verified` for NextAuth v5 adapter, composite PKs on `verification_tokens` and `project_scores`, generated `projects.total_vcus_available`, cascade policies per architecture ¬ß3. All 11 indexes present (2 GIST). Tables owned by `karbonlens` role so future `ALTER TABLE` migrations succeed. Migration file: `scrapers/migrations/001_init.sql`; idempotent under `--single-transaction`.
- **T04 Drizzle schema + DB client** ([feature/T04-drizzle-schema]) ‚Äî TypeScript Drizzle schema at `lib/schema.ts` mirrors all 15 tables from migration 001 (including `users.email_verified`, generated `projects.total_vcus_available`, composite PKs on `project_scores` and `verification_tokens`, `check` constraint on integrity score, PostGIS `geographyPoint` custom type for centroid + satellite-alert location). Singleton client at `lib/db.ts` enforces `DATABASE_URL` at module load. `/api/health` returns sanitized `{ok,db,error}` with HTTP 503 on failure and a classifier that detects auth-failed vs connection-refused without leaking connection strings. `drizzle.config.ts` scoped to introspect-only (SQL files under `scrapers/migrations/` remain authoritative). Post-audit fix round 1: 6 `accounts` token fields renamed to snake_case JS keys to match `@auth/drizzle-adapter` v1.11.2 contract; classifier extended to walk `err.cause.code` for ECONNREFUSED detection.
- **T03 Next.js 15 bootstrap** ([feature/T03-nextjs-bootstrap]) ‚Äî scaffold Next.js 16 (App Router) at the repo root, Tailwind v4 CSS-first with tokens extracted from the legacy prototype, six screens ported with mock data under route groups `(public)` and `(app)`, `netlify.toml` wired for `@netlify/plugin-nextjs`, sole ownership of `.env.example` with eight keys mapped to consuming stories. Static prototype preserved at `legacy/prototype/`. Visual design parity not verified in a browser ‚Äî follow-up needed at T11‚ÄìT18. Route layouts keep `<html>` / `<body>` in root only.
- **T06 Verra scraper** ([feature/T06-verra-scraper]) ‚Äî Python 3.12 scraper ingests Indonesian VCS projects into `projects` + `registries` + `issuances` via Verra's internal OData API (reverse-engineered from the Angular SPA since the documented URLs returned a shell). 64 Indonesian projects populated (>=40 target from PRD); 307 issuances across all vintages; idempotent on re-run. Entity resolution via `pg_trgm` fuzzy matching with a human-in-the-loop `project_match_queue` for 0.70‚Äì0.95 similarity band. Sole owner of `scrapers/pyproject.toml` with all Phase 2 runtime deps (psycopg, httpx, bs4, lxml, shapely, pyproj, pdfplumber, structlog, python-dotenv). Shared helpers at `scrapers/common/` for downstream scrapers to consume. Known limitations: Verra API exposes registration_date not PDD validation_date (T09 scoring will reflect this); status-map covers the 4 common Verra statuses but ~18 projects currently labeled `suspended` when they're in verification ‚Äî follow-up T06.1 will extend the map before T11/T12 frontend. Two match-queue entries pre-resolved as `rejected` (legitimately distinct projects).
- **T10 Regulatory events seed v1** ([feature/v0.1-impl]) ‚Äî 10 hand-curated Indonesian carbon regulations seeded into `regulatory_events` with bilingual summaries (EN + ID), importance classifications (`critical`/`high`/`medium`/`low`), and ad-hoc tags (T15 will build filter UI via `SELECT DISTINCT unnest(tags)`). Idempotent re-apply via `WHERE NOT EXISTS` dedupe; sentinel rows (N/A, TBD) use 3-field keys. `DO $$ ASSERT $$` block in the SQL rolls back the transaction if importance is typo'd. Post-audit fixes landed: row 1 name corrected (Dewan Nasional, not Badan Pengendalian), row 6 tag `mrvb` ‚Üí `mrv`. **Status: `done-pending-factcheck` ‚Äî rows 6‚Äì10 (Permenhut 7/2024, Kepmen LH 20/2025, Perpres 110/2025, Permenhut 6/2026, IDX 2026-07-01 forecast) await Andy's confirmation of document numbers and dates against authoritative sources. Code-correct; fact-correct pending.**
- **T05 NextAuth v5 + Google OAuth** ([feature/T05-nextauth-google]) ‚Äî NextAuth v5 wired to the Drizzle adapter with `strategy: 'database'`, explicit adapter table bindings (`usersTable`, `accountsTable`, `sessionsTable`, `verificationTokensTable`) to match our plural schema names. Protected routes `/projects`, `/prices`, `/regulatory`, `/alerts` redirect unauthed to `/?signin=1`; three public project slugs exempt (`katingan-peatland`, `sumatra-merang-peat`, `rimba-raya`). Onboarding modal collects `persona` + `organization` on first authed render with a 7-day dismiss-snooze cookie. Sign-in and sign-out components, user avatar menu, session type augmentation, and a GCP-setup runbook all in place. **Phase A complete (code, build, middleware, unauthed redirects, API boundary); Phase B (live Google OAuth round-trip including T04's snake_case adapter-token-field verification) pending Andy's GCP credentials ‚Äî see `docs/stories/reports/T05-implementation-report.md` ¬ßPhase-B.**

### Fixed
- **T05 Google provider auto-discovery fix** ([fa51104]) ‚Äî `providers: [Google]` with no args relied on Auth.js v5 auto-detection from `AUTH_GOOGLE_ID` / `AUTH_GOOGLE_SECRET` env vars, but the runbook and `.env.example` use `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`. Result: `client_id=undefined` in the OAuth authorization URL and `invalid_client` from Google on every sign-in attempt. Fixed by passing `clientId` / `clientSecret` explicitly to the `Google(...)` provider. T05 Phase B verified end-to-end on 2026-04-21: user + account + session rows populate correctly; `accounts.access_token` and `accounts.id_token` persist under snake_case JS keys (live validation of T04's adapter-field-name fix). Known residuals: `users.email_verified` not populated by the default adapter mapping; `accounts.refresh_token` empty because the Google provider isn't configured with `access_type=offline`. Neither blocks v0.1; tracked for v0.2.
